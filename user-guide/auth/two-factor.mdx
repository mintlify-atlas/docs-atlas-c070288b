Two-factor authentication (2FA) is one of the most effective defenses against unauthorized access to the OwnPay admin panel. Even if your password is compromised, an attacker cannot log in without physical access to your authenticator app. OwnPay uses the **Time-based One-Time Password (TOTP)** standard, compatible with apps such as Google Authenticator, Authy, and Bitwarden Authenticator.

## Enabling 2FA on Your Account

2FA is configured per account from the **My Account** settings page. The super-administrator should enable it on all privileged accounts.

<Steps>
  <Step title="Open Account Settings">
    Log in to the OwnPay admin panel, then click your avatar or username in the top-right **User Menu** and select **My Account** (or navigate to `/account/settings`).
  </Step>
  <Step title="Locate the Two-Factor Authentication section">
    Scroll to the **Security** section of the account settings page. You will see a **Two-Factor Authentication** toggle currently set to disabled.
  </Step>
  <Step title="Click Enable 2FA">
    Toggle 2FA on. The system will generate a unique TOTP secret and display a **QR code** for you to scan.
  </Step>
  <Step title="Scan the QR code with your authenticator app">
    Open your preferred authenticator app on your mobile device and use its camera/scan function to scan the QR code displayed on screen. The app will add an **OwnPay** entry mapped to your login email.
  </Step>
  <Step title="Verify the setup with a live code">
    Your authenticator app will immediately begin generating 6-digit codes that rotate every 30 seconds. Enter the current code into the **Verification Code** field on the setup screen and click **Confirm**. This proves the pairing succeeded before 2FA is activated.
  </Step>
  <Step title="Store your backup information securely">
    After confirming, save any recovery information provided (such as backup codes or the TOTP secret string) in a secure password manager. You will need these if you lose access to your authenticator app.
  </Step>
</Steps>

<Warning>
  Once 2FA is enabled, it cannot be bypassed at login. If you lose your authenticator device and have no backup codes, a super-administrator must manually revoke and re-enable 2FA on your account from the Staff settings page.
</Warning>

## Using 2FA at Login

Each time you log in with a 2FA-enabled account, you are automatically redirected to the `/2fa` verification screen after entering your password correctly.

<Steps>
  <Step title="Complete the standard login">
    Enter your **Email or Username** and **Password** on the [login page](/user-guide/auth/login) and click **Sign In**.
  </Step>
  <Step title="Open your authenticator app">
    On your mobile device, open the authenticator app and find the **OwnPay** entry.
  </Step>
  <Step title="Read the current 6-digit code">
    Note the active 6-digit code. Each code is valid for a 30-second window. If the timer is nearly expired, wait for the next code to appear before typing — this avoids submitting an expiring code.
  </Step>
  <Step title="Enter the code and verify">
    Type the 6-digit code into the **Verification Code** field on the 2FA screen, then click **Verify**. Upon success, you are redirected to the [Dashboard](/user-guide/dashboard/dashboard).
  </Step>
</Steps>

<Tip>
  If the timer in your authenticator app shows only a few seconds remaining, wait for the next 30-second code to appear before submitting. The system also tolerates up to **60 seconds of clock drift** (2 time slices) to account for minor time desynchronization between your device and the server.
</Tip>

## 2FA Verification Fields Reference

| Field / Option | Required | Description |
|---|---|---|
| **Verification Code** | Yes | The 6-digit numeric code generated by your authenticator app for the current 30-second window. |
| **Verify** | — | Submits the code to complete the login session. |
| **Cancel & Logout** | No | Cancels the authentication flow entirely and returns you to the login page. Use this if you cannot complete 2FA (e.g. on a public computer). |

## How the TOTP System Works

<Accordion title="Clock Drift Tolerance">
  The server accepts codes from the current 30-second time slice as well as the one immediately before and after it — effectively allowing up to **60 seconds** of tolerance. This gracefully handles minor clock differences between your device and the server without compromising security. However, keeping your device time set to **automatic/network time** is strongly recommended.
</Accordion>

<Accordion title="Replay Protection">
  Once a TOTP code is used successfully, it is invalidated immediately and cannot be reused within the same time window. The system tracks the last used time slice to prevent session hijacking via code interception.
</Accordion>

<Accordion title="Rate Limiting">
  Submitting too many invalid codes will trigger the rate limiter and lock you out from further attempts for **5 minutes**. This prevents brute-force attacks against the 6-digit code space.
</Accordion>

## Recovery and Backup Considerations

<CardGroup cols={2}>
  <Card title="Backup Codes" icon="file-shield" href="/user-guide/auth/two-factor">
    Store the TOTP backup codes generated during setup in a secure, offline password manager. These codes allow one-time access if your primary authenticator device is unavailable.
  </Card>
  <Card title="Admin-Assisted Recovery" icon="user-gear" href="/user-guide/auth/login">
    If all backup options are exhausted, a super-administrator can revoke the 2FA secret from the Staff settings page, allowing you to set up a fresh authenticator pairing on your next login.
  </Card>
  <Card title="Multiple Devices" icon="mobile" href="/user-guide/auth/two-factor">
    Some authenticator apps (such as Authy) support multi-device sync. Consider enabling this so you have a fallback device if your primary phone is lost or damaged.
  </Card>
  <Card title="Never Share Your Secret" icon="eye-slash" href="/user-guide/auth/two-factor">
    The TOTP secret key and QR code displayed during setup are unique to your account. Never share them, screenshot them to unsecured locations, or transmit them over unencrypted channels.
  </Card>
</CardGroup>

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---|---|---|
| `Invalid or expired 2FA code` | Device clock is out of sync, or the code expired before submission. | Set your phone's time to **automatic/network time**. Then wait for the next fresh code and enter it quickly. |
| `2FA secret could not be decrypted` | Server decryption key mismatch — typically after a key rotation. | Contact your super-administrator to revoke and re-enable 2FA on your account. |
| `Rate limit exceeded` | Too many incorrect code submissions in a short period. | Wait **5 minutes**, then log in again from the start. |
| Lost authenticator app access | Device lost, factory reset, or app data deleted. | Use a saved backup code or have your super-administrator reset 2FA from the Staff settings page. |
